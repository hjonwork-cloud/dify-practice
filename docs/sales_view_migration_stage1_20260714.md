# 매출 호환 뷰 전체 이관 완료

- 적용일: 2026-07-14
- 신규 뷰: `h_hmfo_fsi_dm.gd_rst_ing.sales_custmasters_compat_v`

## 적용 범위

기존 매출 테이블을 사용하는 실행 코드와 리포트·진단 스크립트를 신규 호환 뷰로 전환했다.

- `T_MAIN`
- `/divisions` 사업부 목록 API
- `/sales` 매출 조회 API
- Dify SQL 생성 안내의 테이블명
- `api/action_signals.py`의 액션 시그널 매출 조회
- `api/forecast_engine_v7.py`의 단일 브랜드 예측 및 직접 실행 분석 쿼리
- 고객 생애주기 Excel 내보내기 스크립트
- 기존 매출 테이블을 직접 참조하던 진단 스크립트

## 호환 뷰 보정

고객마스터의 `FC본부`/`FC본부명`을 `ZC본부`/`ZC본부명`으로 사용한다.

`FC본부`가 비어 있는 경우에는 `ZP본사`로 귀속하지 않고, 거래처 본인 코드와 명칭을 유지하도록 변경했다. 이는 개인형 또는 FC 미지정 거래처가 상위 본사 브랜드 매출에 합산되는 것을 방지한다.

## 스모크 테스트 결과

신규 뷰에서 다음 조회가 정상 동작했다.

- 최신 매출 월: 202607
- 외식식재사업부 월별 매출: 202605~202607
- 브랜드명 검색: 202604 외식3팀 샐러디 3개 ZC 조회
- 영업사원별 매출 집계
- 자재명별 매출 집계

## 전환 대상이 아닌 범위

수익성·미출고 테이블은 매출 메인 테이블과 다른 원천이므로 이번 이관 대상이 아니다.

`자재그룹명`과 `기존자재번호`는 신규 뷰에서 아직 비어 있다. 해당 컬럼을 사용하는 자재그룹 분석은 신규 뷰를 보더라도 결과가 없을 수 있으며, 상품 마스터 조인이 별도로 필요하다.

기존/신규 차이를 확인하는 검증 스크립트 4개는 비교 목적상 기존 테이블 참조를 의도적으로 유지한다.

## 롤백

문제가 발생하면 각 실행 모듈의 매출 테이블 상수를 기존 `h_hmfo.gd_dcube.01_sap_sales_custmasters`로 되돌리면 된다.
