## 2. 챗봇 Dify 우회/바이패스 관련 수정

### 2.1 브랜드 매출 질의가 Dify로 넘어가던 문제

사용자가 브랜드 매출 질의가 Dify로 넘어간다고 보고했다.

대표 사례:

- `신화푸드 매출액`
- 월을 명시하지 않은 브랜드 매출 질의

확인 결과, 월 미명시 브랜드 매출 로직에서 당월 데이터가 없고 fuzzy candidate도 없을 때 `pass`로 흘러가면서, 이후 로직에서 브랜드명을 영업사원명처럼 해석해 Dify로 넘기는 문제가 있었다.

### 조치

- 당월 데이터가 없으면 전월 데이터로 fallback하고 즉시 return하도록 수정
- Dify로 fall-through되지 않도록 처리

커밋:

- `b1aaaae` — 월 미명시 브랜드 매출 당월 데이터 없음 시 전월 fallback 및 return 수정

---

### 2.2 전월 fallback 카드 문구 혼선 수정

당월 데이터가 없어 전월 데이터로 fallback했는데, 응답 카드에는 `이번달 누계/예상`처럼 표시되어 혼선이 있었다.

### 조치

- fallback 시 `_build_brand_forecast_card`를 사용하지 않고 단순 전월 기준 카드로 표시
- 예: `5월 매출액 ... ※ 6월 데이터 미적재 — 5월 기준`

커밋:

- `eceac6d` — 브랜드 매출 전월 fallback 시 forecast 카드 대신 단순 포맷 사용

---

### 2.3 `사업부 5월 매출` 질의가 Dify로 넘어가던 문제

사용자가 아래 질의 차이를 보고했다.

- `외식식재사업부 5월 매출액` → 정상 바이패스
- `사업부 5월 매출액` → Dify로 넘어감

원인은 월별 총매출 정규식이 `사업부` 앞에 명시적 텍스트가 있어야 매칭되도록 되어 있었기 때문이다.

### 조치

- 단독 `사업부` 키워드를 `외식식재사업부`로 정규화한 뒤 월별 총매출 패턴 매칭

커밋:

- `4370bfb` — 사업부 단독 키워드를 외식식재사업부로 정규화

`h_hmfo_fsi.gd_rst_ing`

- 생성 대상 스키마를 사용자가 요청한 `h_hmfo_fsi_dm.gd_rst_ing`로 변경

## 7. Databricks에서 뷰 이름 변경 방법 안내

사용자가 `h_hmfo_fsi_dm.gd_rst_ing.chatbot_sales_custmasters_compat_v` 이름 변경 위치를 문의했다.

안내한 방법:

1. Databricks 접속
2. 왼쪽 메뉴 `Catalog`
3. Catalog: `h_hmfo_fsi_dm`
4. Schema: `gd_rst_ing`
5. View: `chatbot_sales_custmasters_compat_v`
6. 우측 상단 또는 더보기 메뉴에서 `Rename`

SQL 방식:

```sql
ALTER VIEW h_hmfo_fsi_dm.gd_rst_ing.chatbot_sales_custmasters_compat_v
RENAME TO h_hmfo_fsi_dm.gd_rst_ing.새이름;
```

대안:

```sql
CREATE OR REPLACE VIEW h_hmfo_fsi_dm.gd_rst_ing.새이름 AS
SELECT *
FROM h_hmfo_fsi_dm.gd_rst_ing.chatbot_sales_custmasters_compat_v;

DROP VIEW h_hmfo_fsi_dm.gd_rst_ing.chatbot_sales_custmasters_compat_v;
```

## 8. 신규/기존/중단 거래처 판단 기준 정리

### 8.1 최초 제안 기준과 수정

처음에는 일반적인 기준으로 아래를 제안했다.

- 신규: 기준연도에 매출 있고 이전 전체 이력 없음
- 기존: 기준연도에 매출 있고 이전 매출 이력 있음
- 중단: 전년도 매출 있고 기준연도 매출 없음

사용자가 현재 챗봇의 신규 매출액 기준은 `전년 10월 이후 최초 매출 발생`이라고 정정했다.

### 8.2 최종 확정 기준

기준연도 `Y` 기준:

| 구분 | 기준 |
|---|---|
| 신규 | `Y`년에 매출 있음 + 최초 매출일이 `Y-1년 10월 1일` 이후 |
| 기존 | `Y`년에 매출 있음 + 최초 매출일이 `Y-1년 10월 1일` 이전 |
| 중단 | `Y-1`년에 매출 있음 + `Y`년에 매출 없음 |

예: 2026년 기준

| 구분 | 조건 |
|---|---|
| 신규 | 2026년 매출 있음 + 최초 매출일 `>= 2025-10-01` |
| 기존 | 2026년 매출 있음 + 최초 매출일 `< 2025-10-01` |
| 중단 | 2025년 매출 있음 + 2026년 매출 없음 |

사용자는 중단 기준을 `2025년 매출 있음 + 2026년 매출 없음` 방식으로 확정했다.

   - `ZC코드`, `ZC코드명`은 `개인형`

## 12. 현재 미완료/주의 사항

### 12.1 운영 코드 전환 미완료

아직 운영 챗봇 코드는 신규 호환 뷰로 전환하지 않았다.

아래 상수들은 여전히 기존 테이블을 바라본다.

- `api/main.py`의 `T_MAIN`
- `api/action_signals.py`의 `T_MAIN`
- `api/forecast_engine_v7.py`의 `T_MAIN_FE`

운영 전환 시 추가로 Dify SQL prompt 내 테이블명도 변경해야 한다.

---

### 12.2 `ZC본부명` 공백 이슈

호환 뷰에서 `ZC본부명`은 현재 공백이다.

영향:

- 브랜드명 기반 쿼리에서 `ZC본부명 LIKE '%샐러디%'` 같은 질의는 신규 호환 뷰만으로는 작동하지 않음
- 코드 기반 `ZC본부 IN (...)` 방식은 가능

향후 필요:

- IT에 `ZC본부명` 원천 컬럼 요청
- 또는 별도 매핑 테이블 확보

---

### 12.3 상품/자재 관련 컬럼 미보강

아래 컬럼은 현재 `NULL`이다.

- `자재그룹`
- `자재그룹명`
- `기존자재번호`

향후 필요:

- 상품/자재 마스터 확보 후 join 보강

---

### 12.4 미출고 테이블 전환 미구현

`46_helo_periodic_unshipped` 대체 후보인 아래 테이블 검토는 했지만 운영 코드 전환은 아직 하지 않았다.

`h_hmfo_fsi.gd_fsi_ent.helo_periodic_unshipped_hist_f`

현재 판단:

- 대체 가능성은 비교적 높음
- `총미출수량` 등 신규 컬럼 기준으로 쿼리 수정 필요
