# 미출고 FSI 호환 뷰 이관 완료

- 적용일: 2026-07-14
- 기존 테이블: `h_hmfo.gd_dcube.46_helo_periodic_unshipped`
- 신규 원천: `h_hmfo_fsi.gd_fsi_ent.helo_periodic_unshipped_hist_f`
- 신규 호환 뷰: `h_hmfo_fsi_dm.gd_rst_ing.unshipped_compat_v`

## 적용 내용

신규 FSI 미출고 원천과 고객마스터 `sap_zsdrxd03_customer_master_rst_ing_d`를 조인한 호환 뷰를 만들고, 아래 실행 모듈의 `T_MISULGO`를 전환했다.

- `api/main.py`: 영업사원·팀 미출고 조회
- `api/action_signals.py`: 브랜드 미출고 다발 시그널

원천의 문자열 `미출수량` 대신 수치형 `총미출수량`을 호환 뷰의 수치형 `미출수량` 및 `미출고수량`으로 제공한다.

브랜드 시그널 호환용 컬럼도 제공한다.

- `거래처명` = `통합배송처명`
- `자재명` = `상품명`
- `주문일자` = `출고일자`
- `ZC본부` / `ZC본부명` = 고객마스터의 `FC본부` / `FC본부명`, 미매핑 시 배송처 본인 값

## 검증 결과

- 신규 원천 최신 출고일: 2026-07-14
- 기존 테이블 최신 출고일: 2026-05-27
- 신규 원천은 더 최신 데이터를 제공한다.
- 영업사원별 미출고 조회, 팀별 미출고 조회, 생활맥주 브랜드 미출고 집계가 신규 호환 뷰에서 정상 동작했다.

## 주의 사항

최신일 기준 `통합배송처ID`와 고객마스터 코드의 직접 조인율은 약 49%였다. 따라서 브랜드 `ZC본부명` 기반 미출고 집계는 고객마스터에 매핑된 배송처를 정확히 집계하며, 미매핑 배송처는 배송처명 자체를 `ZC본부명`으로 사용한다.

기존과 신규의 과거 일자별 행 수·미출 수량은 동일하지 않다. 신규 FSI 원천의 `총미출수량`을 챗봇의 운영 기준으로 사용한다.

## 롤백

문제가 발생하면 `api/main.py`와 `api/action_signals.py`의 `T_MISULGO`를 기존 테이블로 되돌린다.
